# שילוב האתר ב‑Base44

האתר נכתב כך שיעבוד **גם בלי Base44** (נתונים סטטיים מתוך `index.html`) **וגם איתו** —
ברגע ש‑`window.base44` קיים בעמוד, הפרויקטים נטענים מה‑entity והטופס שומר ליד.
אין צורך לשנות את קוד העיצוב; רק להוסיף את שכבת הנתונים.

## מה כבר מוכן בקוד (`index.html`)
- `loadProjects()` — טוען מ‑`base44.entities.Project.list()`, וברירת מחדל לרשימה הסטטית אם אין SDK/נתונים.
- כרטיס הדגל נקבע מהרשומה עם `featured: true`.
- טופס יצירת הקשר — שומר `base44.entities.Lead.create(...)`, ובכשל נופל ל‑`mailto`.
- כל השדות תואמים בדיוק לסכמות שבתיקייה הזו.

## שלבים

### 1. צרו אפליקציה ב‑Base44
פתחו אפליקציה חדשה (React). זה ייצור גם `window.base44` (ה‑SDK) בתוך העמוד.

### 2. צרו שני entities
- **Project** — העתיקו את הסכמה מ‑[`Project.schema.json`](./Project.schema.json)
- **Lead** — העתיקו את הסכמה מ‑[`Lead.schema.json`](./Lead.schema.json)

(ב‑Base44: Data → New Entity → הדביקו את ה‑JSON, או הגדירו את השדות ידנית לפי הסכמה.)

### 3. העלו תמונות ל‑File Storage
העלו את התמונות מתיקיית `assets/` (למשל `proj-keren.jpg`, `har-avital/day-web.jpg`,
וקבצי הבניין `assets/building/day.webp` + `night.webp`), והעתיקו את ה‑URLs.

### 4. הזריקו את נתוני הפרויקטים
ייבאו את [`projects.seed.json`](./projects.seed.json) ל‑entity **Project**
(5 רשומות: הר אביטל מסומן `featured`). **החליפו את שדות `image`** ב‑URLs מהשלב הקודם.

### 5. שילוב העיצוב הקיים
שתי דרכים:
- **הטמעה כקוד מותאם** — הדביקו את תוכן `index.html` (ה‑markup + ה‑`<style>` + ה‑`<script>`)
  לתוך עמוד/רכיב "Custom Code / HTML" ב‑Base44. כיוון ש‑`window.base44` כבר קיים שם,
  הפרויקטים והטופס יתחברו אוטומטית.
- **רכיב React** — עטפו את ה‑markup ברכיב והזריקו את ה‑HTML; שמרו על ה‑`id`‑ים כפי שהם
  כדי שה‑JS ימצא את האלמנטים.

### 6. אוטומציה (מומלץ)
ב‑entity **Lead** הוסיפו אוטומציה: *On create → Send email* אל `Office@shellyurban.co.il`
כדי לקבל כל פנייה גם במייל (בנוסף לשמירה ב‑DB).

## מבנה רשומת Project (לעריכה ידנית)
| שדה | ערך | הערה |
|-----|-----|------|
| `cat` | `renewal` / `sale` / `build` | קובע את האנימציה בליד (התחדשות = סוויפ ישן→חדש, שיווק = פעימה) |
| `stage` | `1` / `2` / `3` | רישוי / ביצוע / אכלוס — ממלא את פס‑הסטטוס |
| `facts` | מערך של `{k,v,l}` | `k` ∈ units/floors/renew/build/park/key (אייקון) |
| `featured` | `true` בפרויקט אחד | הכרטיס הגדול בראש הגריד |
| `order` | מספר | סדר הצגה |

> הערה: הנתונים נאספו מהאתר הרשמי shellyurban.co.il. עדכנו מספרים/סטטוסים בפאנל של Base44 בכל עת — האתר יתעדכן אוטומטית.

## מיילים, אישור ממותג ואינטגרציה
- **`email-confirmation.html`** — תבנית מייל אישור ממותגת (לוגו + צבעי מותג) לשליחה אוטומטית לפונה.
- **`Unit.schema.json`** — entity דירה (בסיס ללוח זמינות ולסנכרון "נמכר/פנוי" מול תוכנת הניהול).
- **`INTEGRATION.md`** — ארכיטקטורת החיבור המלאה: מיילים, דפי-פרויקט אוטומטיים, וסנכרון סטטוס דירות עם תוכנת הניהול דרך ה-REST API של Base44.

> הטופס עודכן: נוסף שדה **`email`** (אופציונלי) כדי לאפשר שליחת אישור ממותג לפונה.

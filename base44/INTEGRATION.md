# אינטגרציה: שתי אפליקציות Base44 (ניהול ⇄ אתר)

מודל מאושר: **שתי אפליקציות Base44 נפרדות**.

| אפליקציה | תפקיד | Entities (מקור אמת) |
|---|---|---|
| **App: ShellyUrban** (ניהול) | מקור האמת לפרויקטים ולדירות | `Project`, `Unit` (סטטוס נמכר/פנוי) |
| **App: Website** (האתר הזה) | מציג פרויקטים/זמינות, אוסף לידים, שולח מייל אישור | `Project`, `Unit`, `Lead` |

```
┌────────────────────────┐   automation: REST push    ┌────────────────────────┐
│  App: ShellyUrban       │ ─────────────────────────▶ │  App: Website           │      ◀── גולשים
│  (ניהול — מקור אמת)      │   Project / Unit            │  (תצוגה)                │
│  Project, Unit          │                            │  Project, Unit, Lead    │
└────────────────────────┘ ◀───(אופציונלי) Lead push ── │  Lead → מייל אישור ממותג │
                                                        └────────────────────────┘
```

**כיוון הסנכרון: ניהול → אתר (push).** האתר הציבורי קורא **רק מה-DB של עצמו** — לעולם לא מחזיק מפתחות API.

---

## 1. סנכרון פרויקטים ודירות (ניהול → אתר)
באפליקציית **ShellyUrban** מגדירים אוטומציות שמושכות/דוחפות לאתר דרך ה-REST API של אפליקציית האתר:

- **On `Project` create/update → upsert** ב-Website (התאמה לפי `external_id` = מזהה הפרויקט בניהול).
  → פותחים פרויקט בניהול ⇒ **דף הפרויקט מופיע באתר אוטומטית.**
- **On `Unit` update (status) → patch** ב-Website (התאמה לפי `external_id` = מזהה הדירה בניהול).
  → מסמנים דירה "נמכרה" בניהול ⇒ **האתר מסמן נמכר.**

קריאות REST (מתוך האוטומציה באפליקציית הניהול):
```
# פרויקט חדש/עודכן
POST/PATCH https://app.base44.com/api/apps/{WEBSITE_APP_ID}/entities/Project
Authorization: Bearer {WEBSITE_API_KEY}
{ "external_id":"<id בניהול>", "title":"...", "location":"...", "cat":"build", "stage":1, "image":"<URL>", "facts":[...] }

# עדכון סטטוס דירה
PATCH https://app.base44.com/api/apps/{WEBSITE_APP_ID}/entities/Unit/{id}
Authorization: Bearer {WEBSITE_API_KEY}
{ "status":"sold" }
```
> `external_id` הוא **מפתח ההתאמה** — מבטיח שעדכון מהניהול יפגע ברשומה הנכונה באתר (בלי כפילויות).

## 2. לידים ומייל אישור (באפליקציית האתר)
- טופס האתר יוצר `Lead` (כבר מחובר בקוד).
- אוטומציה ב-Website: **On `Lead` create →** (א) Send Email התראה ל-`Office@shellyurban.co.il`; (ב) Send Email אישור ממותג לפונה מתוך [`email-confirmation.html`](./email-confirmation.html) (אם נמסר `email`).
- **אופציונלי:** אוטומציה נוספת שדוחפת את הליד גם לאפליקציית הניהול (CRM) דרך ה-API שלה.

## 3. אבטחה
- `WEBSITE_API_KEY` נשמר **רק בצד אפליקציית הניהול** (באוטומציה שרצה בשרת) — **לעולם לא בקוד האתר הציבורי**.
- מומלץ מפתח ייעודי עם הרשאת כתיבה ל-`Project`/`Unit` בלבד.

---

## מה צריך כדי לממש
1. **`WEBSITE_APP_ID` + `WEBSITE_API_KEY`** של אפליקציית האתר (להזין באוטומציות של אפליקציית הניהול).
2. להוסיף שדה **`external_id`** ל-`Project` (כמו שכבר קיים ב-`Unit`) — מפתח ההתאמה.
3. **גישה ל-`AvivShelly48/shellyurban`** (או הדבקת שמות ה-entities/שדות שלה) — כדי שאמפה 1:1 בין שדות הניהול לשדות האתר ואכין את גוף האוטומציות המדויק.

> בלי גישה לאפליקציית הניהול אני יכול להכין את **צד האתר** במלואו (entities, טופס, מיילים) ואת **מפרט האוטומציות** — אך מיפוי השדות הסופי דורש לראות מה ה-entities של הניהול.

# אינטגרציה: אתר ⇄ Base44 ⇄ תוכנת הניהול

מסמך זה עונה על שלוש המטרות: **(1)** מיילים + אישור ממותג, **(2)** דף-פרויקט שנוצר אוטומטית כשנפתח פרויקט בתוכנת הניהול, **(3)** סימון "נמכר/פנוי" שמסתנכרן מהתוכנה לאתר.

## העיקרון
Base44 הוא ה-**Backend** של האתר: מסד נתונים (entities), אוטומציות, ו-**REST API** לכל אפליקציה. האתר (`index.html`) כבר קורא מה-entities (`Project.list()`) ושומר לידים (`Lead.create()`). לכן כל "מקור האמת" הוא Base44 — ותוכנת הניהול רק צריכה לכתוב אליו.

```
תוכנת הניהול  ──(REST API / webhook)──▶  Base44 (entities)  ──◀  האתר קורא ומציג
```

---

## 1. מיילים + אישור ממותג  ✅ (מוכן להגדרה)
ב-Base44, על entity **Lead**, מגדירים שתי אוטומציות **On create**:

1. **התראה פנימית** → *Send Email* אל `Office@shellyurban.co.il`
   נושא: `ליד חדש מהאתר — {{name}}` · גוף: `{{name}} · {{phone}} · {{project}} · {{message}}`.
2. **אישור ממותג לפונה** → *Send Email* עם גוף ה-HTML מתוך [`email-confirmation.html`](./email-confirmation.html)
   (משתני מיזוג `{{name}}`, `{{project}}`). *דורש איסוף שדה `email` בטופס — ראו הערה למטה.*

> **הערה:** הטופס הנוכחי אוסף שם + טלפון בלבד. כדי לשלוח אישור **במייל** צריך להוסיף שדה `email` לטופס ול-`Lead.schema`. אם רוצים אישור ב-**וואטסאפ** במקום — ראו §4.

---

## 2. דף-פרויקט אוטומטי כשנפתח פרויקט בתוכנה
פרויקטים באתר נטענים מ-entity **Project**. לכן:
- כשנפתח פרויקט חדש בתוכנת הניהול → היא יוצרת רשומת **Project** ב-Base44 (REST API `POST`).
- האתר מציג אותו אוטומטית בגריד, ללא שינוי קוד.

דוגמת קריאה מהתוכנה (REST):
```
POST https://app.base44.com/api/apps/{APP_ID}/entities/Project
Authorization: Bearer {API_KEY}
Content-Type: application/json

{ "title":"...", "location":"...", "cat":"build", "stage":1, "image":"<URL>", "facts":[...] }
```

---

## 3. סנכרון "נמכר / פנוי"
מודל הנתונים: entity **Unit** ([`Unit.schema.json`](./Unit.schema.json)) — דירה בודדת עם `status` ו-`external_id` (מזהה הדירה בתוכנה, מפתח ההתאמה).
- כשמסמנים דירה כ"נמכרה" בתוכנה → היא מעדכנת `PATCH` של ה-Unit המתאים (`status: "sold"`).
- לוח הזמינות באתר (כשנבנה) יציג לפי `status`: 🟢 פנוי · 🟡 בהמתנה · 🔴 נמכר.

```
PATCH https://app.base44.com/api/apps/{APP_ID}/entities/Unit/{id}
Authorization: Bearer {API_KEY}
{ "status": "sold" }
```
> לוח הזמינות החזותי — "לא עכשיו" לפי בקשתך; המודל והנתיב מוכנים לכשנרצה.

---

## 4. אופן החיבור של תוכנת הניהול (`AvivShelly48/shellyurban`)
תלוי במה התוכנה:
- **אם היא עצמה אפליקציית Base44** → אפשר לאחד entities (אותו DB) או לסנכרן בין אפליקציות ב-automation. **הכי חלק.**
- **אם היא חיצונית עם API** → היא קוראת ל-REST API של Base44 (§2-3) ב-push בכל שינוי.
- **אם אין לה API** → גשר ביניים (Airtable/Google-Sheet) + GitHub Action/Make שמסנכרן.

**וואטסאפ אישור אוטומטי:** דורש WhatsApp Business API (Twilio/360dialog/Meta) — אוטומציה ב-Base44 שקוראת ל-API בעת יצירת Lead. חלופה מיידית: כפתור קליק-לוואטסאפ (`wa.me`).

## אבטחה
- מפתח ה-API של Base44 נשמר רק בצד תוכנת הניהול/השרת — **לעולם לא בקוד האתר** (האתר ציבורי).
- מומלץ מפתח ייעודי עם הרשאות מינימליות (כתיבה ל-Project/Unit בלבד).
